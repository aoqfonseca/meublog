--- layout: post title: "Usando SVN como reposit\xC3\xB3rio de c\xC3\xB3digo" tags: - "Inform\xC3\xA1tica" status: publish type: post published: true meta: {} --- Estou começando um projeto e gostaria muito de utilizar o GIT como repositório de código por motivos de velocidade, facilidade e etc. A questão que, infelizmente, não consegui encontrar nenhuma versão DAEMON para windows. Bom, aqui cabe uma explicação: sei que alguns vão comentar que eu sou lixo por estar usando windows e não linux... blablabla.... Por minha escolha estaria usando linux no servidor mas a questão é que a aplicação será feita em .NET e gostaria de colocar continuos integration, build automático, entre tantas coisas mais neste e sei que linux é trabalhoso fazer para uma aplicação windows (poderia usar o wine, mas aqui preciso distribui-la para windows depois... ) Devido a esta dificuldade de encontrar uma versão do GIT para windows, me voltei para meu bom e velho amigo SVN. Instalei o VisualServer (muito bom para o pessoal que gosta de telas gráficas para administrar as coisas). Feito isso foi só criar os usuários (optei por não usar a autorização do windows pois como uso Novell, desconheço como integrar o server ao ldap dele... ) e criar um repo: Primeira dica, nunca crie nada no diretorio raiz, não é uma boa prática. Sugiro sempre criar um repositorio e partir dele sair criando o que deseja. Para criar um repo, a partir da ferramenta é fácil - busquem pelo tutorial na internet. Pela linha de comando foi apenas svnadmin create [repo] O repo do comando acima é o path do repositorio (ele será criado dentro do seu servidor) Depois disso eu uso outro comando para verificar se o status dele está ok e pronto para uso svnadmin verify [nome do repositorio que deseja ver] Agora no resto ir numa máquina cliente e importar para nosso controlador de versão os arquivos que desejamos versionar: Na máquina cliente, digite o comando abaixo para se conectar ao servidor e importar os arquivos: svn import [caminho do diretorio onde estão os arquivos a importar] [url do servidor com o repositorio] Ex: svn import MyProject http://123.123.123.4:8443/svn/MyRepo/trunk Isso importara os arquivos. Depois (eu desconheço outro jeito) e faço o checkout do projeto e partir dai tenho um projeto local ligado ao repositorio SVN svn co [url repo] svn co https://blabla:8443/svn/MyRepo/Trunk Isso fara funcionar. Existem opções de ferramentas para fazer isso de forma mais visual. A que mais recomendo é a TortoiseSVN. Ele se integra ao menu contexto do gerenciador de arquivo e fica bastante simples trabalhar. Outra opções são os plugins paras os builder .
