--- layout: post title: Instalando o Cruise Control .NET e resolvendo alguns problemas tags: - "Inform\xC3\xA1tica" status: publish type: post published: true meta: {} --- Passo a passo para a instalação do Cruise Control .NET em um servidor windows 2003, com IIS. .NET 2.0 e cujo repositório está em um SVN (Subversion Control Version): 1 - Realizar o download da instalação no site do projeto 2- Baixar a instalação do NUnit para poder realizar os teste automáticos do sistema 3- Após isso, logar no servidor com o administrador. Uma coisa importante é verificar a sua versão da SDK do .NET. Para as versões mais recentes do CCnet é preciso .NET 2.0 ou mais atual para funcionar 4 - Executar o instalador. Normalmente ele fará tudo por você ( inclusive criará o serviço do windows e o diretorio virtual dentro do IIS para a webdashboard) 5 - Uma dica importante é ir no serviço criado e configurá-lo para executar na conta local ou na conta administrador 6 - Ir na pasta onde instalou o CCnet ( geralmente Program Files/ Cruise Control). Dentro da pasta ir para a pasta server e editar o arquivo ccnet.config. Este arquivo será onde definirá os projetos e demais configurações para a integração desejada. 7 - Na pasta onde definiu o seu workdiretory, realizar o checkout do projeto e aceitar o certificado emitido de forma permanente. Assim o sistema irá gravá-lo e os builds do ccnet vão funcionar
